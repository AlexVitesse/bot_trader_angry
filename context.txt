PROJECT_NAME: Binance Futures Scalper (Python)
TYPE: Algorithmic Trading (Mean Reversion + Trend Filter)
TARGET EXCHANGE: Binance Futures
TARGET PAIR: BTC/USDT
CAPITAL: $100 USD (Recomendado para esta configuracion)
LEVERAGE: 10x
BASE_ORDER_MARGIN: $12

STRATEGY: "Smart Metralladora v6.7"
- Timeframe: 1m (High Frequency)
- Indicators: EMA 200 (Trend Filter), Bollinger Bands (20, 2.0), StochRSI (14, k=3).
- Entry LONG: Price > EMA 200 + rebote BB Lower + StochRSI < 20.
- Entry SHORT: Price < EMA 200 + rechazo BB Upper + StochRSI > 80.
- Management: DCA (Grinder) - Max 2 Safety Orders (Martingale x2).
- Take Profit: 0.6% desde promedio.
- Stop Loss: 1.5% desde promedio (catastrofico).
- Execution: Real-time Websocket (websocket-client) + REST API (requests).

TECH STACK:
- Language: Python 3.12+
- Connectivity: REST API (requests + HMAC signing), Binance Websockets.
- Data Storage: Parquet (Historical via fetch_history.py).
- Backtesting: Custom vectorized backtest (backtest_metralladora.py).

CRITICAL CONSTRAINTS:
1. Minimum Order Size: Binance requires ~$100 notional. With $12 margin * 10x = $120 notional.
2. Risk Tolerance: Experimental Phase (Paper Trading en Testnet).
3. Latency: Local execution. Acknowledge network delays (150-300ms).
4. No Manual Intervention: Bot must handle Entry, Stop Loss, Take Profit, and DCA.
